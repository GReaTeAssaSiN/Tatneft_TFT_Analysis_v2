# Tatneft TFT Analysis

Система прогнозирования продаж топлива и товаров магазина на АЗС сети Татнефть на основе нейросетевой модели временных рядов — Temporal Fusion Transformer.

Проект реализован в виде трёх интерактивных веб-приложений (дашбордов), которые запускаются в браузере и не требуют написания кода для работы с данными.

---

## Зачем это нужно

АЗС — это не просто заправка. Каждая станция продаёт несколько видов топлива, работает магазин, кафе. Объём продаж зависит от дня недели, погоды, праздников, акций, трафика на дороге, действий конкурентов и десятков других факторов. Предсказать, сколько литров солярки уйдёт в следующую неделю на конкретной трассовой АЗС — нетривиальная задача.

Эта система позволяет:
- загрузить исторические данные по всем станциям сети,
- подготовить их для обучения нейросети (очистить, нормализовать, настроить признаки),
- обучить модель TFT, которая учится на паттернах прошлого и делает прогноз на будущее,
- визуально исследовать данные и результаты прогноза в удобном интерфейсе.

## Как это работает — три шага

### Шаг 1 — Подготовка данных

```bash
streamlit run dashboard/data_prep_dashboard.py
```

Открывается дашборд с пятью вкладками:

- **Файлы** — загружаем исходные CSV-файлы (статика по станциям + временной ряд продаж и цен). Можно также восстановить ранее сохранённую сессию.
- **Объединение** — настраиваем, как таблицы склеиваются между собой (по каким ключам).
- **Препроцессинг** — для каждой колонки выбираем метод обработки: нормализация, кодирование категорий, логарифмирование и т.д. Можно добавить формульные колонки, извлечь части даты.
- **Сплит** — делим данные на обучающую, валидационную и тестовую выборки по датам.
- **TFT + Экспорт** — указываем, какая колонка время, какая — группировка (номер станции), что предсказываем (таргеты), а что является признаками. Сохраняем всё в папку `exports/`.

Результат — готовая **сессия** в папке `exports/<название>/`: обработанные данные, настройки и параметры для обратного пересчёта.

### Шаг 2 — Обучение и прогноз

```bash
streamlit run dashboard/train_dashboard.py
```

- Выбираем сохранённую сессию. Дашборд автоматически проверяет, что все нужные файлы на месте и корректны.
- На вкладке **Конфигурация** задаём гиперпараметры модели: размер батча, количество эпох, размерность скрытых слоёв, горизонт прогноза и т.д. Есть готовые пресеты для CPU и GPU.
- На вкладке **Обучение** нажимаем «Запустить» — лог обучения стримится прямо в браузер в реальном времени. Обучение можно остановить.
- На вкладке **Результаты** смотрим графики потерь (train/val loss), видим когда сработал early stopping.
- На вкладке **Прогнозы** выбираем чекпоинт (лучшая модель или конкретная эпоха), запускаем прогнозирование, скачиваем CSV с результатами (фактические значения + прогноз с доверительными интервалами).

### Шаг 3 — Аналитика

```bash
streamlit run dashboard/analytics_dashboard.py
```

Независимый дашборд для исследования данных и результатов. Работает с любой сохранённой сессией.

- **EDA-анализ** — динамика продаж по времени, разбивка по видам топлива, категориям магазина, сезонность, фильтрация по станциям и датам.
- **Статистика** — сравнение станций между собой, распределения, корреляции.
- **Прогнозы** — сравнение фактических и прогнозных значений по каждому таргету и станции.
- **Рекомендации** — выводы на основе данных.

## Установка

Требования: Python 3.10+, Windows/Linux/macOS, ~4 ГБ свободного места (зависимости).

```bash
git clone https://github.com/<your-username>/Tatneft_TFT_Analysis_v2.git
cd Tatneft_TFT_Analysis_v2

python -m venv venv
venv\Scripts\activate       # Windows
# source venv/bin/activate  # Linux / macOS

pip install -r requirements.txt
```

После установки запустите любой из трёх дашбордов командой выше. Браузер откроется автоматически.

> Обучение на CPU работает, но медленно — для больших объёмов данных рекомендуется GPU с поддержкой CUDA.

## Обученная модель

В репозитории включены результаты обучения модели на синтетических данных АЗС Татнефть (сессия `ActualData_2026-05-30`):

| Файл | Описание |
|------|----------|
| `training/.../train_config.json` | Гиперпараметры, с которыми обучалась модель |
| `training/.../checkpoints/*.ckpt` | Сохранённая нейросеть (веса модели) |
| `training/.../predictions/*.csv` | Готовые прогнозы на тестовой и валидационной выборках |
| `training/.../logs/` | Логи обучения для TensorBoard |

Это означает, что можно сразу открыть дашборды аналитики и прогнозов и посмотреть на результаты — без необходимости обучать модель заново.

---

## Данные

Исходные данные **не включены** в репозиторий. Система ожидает два CSV-файла:

- **Статика** — характеристики станций: тип дороги, инфраструктура, количество колонок по видам топлива, конкуренты в радиусе.
- **Временной ряд** — ежедневные данные по каждой станции: объёмы продаж, розничные цены, цены конкурентов, погода, трафик, активность акций.

## Структура репозитория

```
dashboard/        — три Streamlit-дашборда
utils/            — логика обработки данных и обучения (без UI)
tft/              — скрипты обучения и прогнозирования
exports/          — тестовые сессии предобработки (реальные данные Татнефть не включены)
training/         — артефакты обучения: модель, прогнозы, логи
requirements.txt  — зависимости Python
```
